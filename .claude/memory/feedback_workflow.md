---
name: 開発着手前のドキュメント作成ワークフロー
description: 実装前に要件定義書・設計書・ADR を必ず作成するというユーザーの作業スタイル
type: feedback
---

実装を始める前に、必ず以下の順序でドキュメントを揃える。

1. 不明点をユーザーに質問して仕様を固める
2. `docs/requirements.md`（要件定義書）を作成
3. `docs/design.md`（設計書）を作成
4. `docs/adr/ADR-NNN-*.md`（ADR）を作成
5. `CLAUDE.md` を作成・更新
6. 実装へ進む

**Why:** ユーザーは「最初からしっかり設計する」方針を明示している。設計なしに実装を始めるとあとで破綻しやすい。

**How to apply:** 新機能・新プロジェクトの依頼が来たらまず質問リストを提示し、回答を得てからドキュメント作成→実装の順で進める。ドキュメントを飛ばして実装を提案しない。
