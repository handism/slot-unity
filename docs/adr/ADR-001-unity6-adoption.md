# ADR-001: Unity 6 の採用

**ステータス**: 承認済み
**日付**: 2026-03-20

---

## コンテキスト

スロットゲームの開発エンジンを選定する。要件は PC（Windows / macOS）向け、1人開発、長期保守性の確保。

## 決定

**Unity 6.3 LTS を採用する。**

## 理由

- ユーザー指定の要件
- Unity 6 では `Awaitable` API により async/await が標準化されており、リールアニメーション等の非同期制御に適している
- 2D UI 開発において豊富なエコシステム（UGUI / TextMeshPro / DOTween）が利用可能
- 1人開発での学習コストが低い（既存ナレッジの活用）

## トレードオフ

- Godot 4 は軽量で学習コストが低いが、アセットエコシステムが Unity より薄い
- Unreal Engine 5 はビジュアル品質が高いが、2D スロットには過剰スペック
