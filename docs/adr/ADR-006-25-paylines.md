# ADR-006: ペイラインを 25 固定ラインとする

**ステータス**: 承認済み
**日付**: 2026-03-20

---

## コンテキスト

5 リール・3 行のグリッドにおいて、ペイラインの方式を決定する。

## 選択肢

| 方式 | 概要 |
|------|------|
| **25 固定ライン** | 25 本のラインを常に有効 |
| 243 Ways | 全シンボル組み合わせを判定（ライン概念なし） |
| 9 / 15 ライン（可変） | プレイヤーがライン数を選択可能 |
| 1024 Ways | 4 行グリッド向け |

## 決定

**25 本の固定ペイライン（常時有効）を採用する。**

## 理由

- 5 リール × 3 行のグリッドで最も一般的な構成。プレイヤーにとって直感的
- 25 固定により「1 スピンのコスト = ベット額」でシンプルに計算できる（243 Ways は 1 スピン = ライン数 × 単価 で複雑）
- ライン数可変にすると UI・配当計算・RTP 管理が複雑化するため、1 人開発では固定が現実的
- `PaylineData` ScriptableObject に定義するため、将来的なライン数変更もコード修正なしで対応可能

## トレードオフ

- 243 Ways に比べてヒット頻度が低く感じられる場合がある
  → リールストリップの重み調整で体感ヒット率を 30〜40% 程度に調整して対応
- プレイヤーがラインを選べないため選択の自由度がない
  → アーケード向けではシンプルさが優先されるため許容
